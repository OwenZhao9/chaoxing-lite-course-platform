# 大学英语智课通使用文档

本文档用于交付演示人员照着操作。项目是一个面向“大学英语”课程的简化版学习通平台原型，包含首页登录、管理员、教师、学生三种角色，以及本地 Express API、React 前端和已接入 DeepSeek 的 AI 助教接口。

## 1. 环境准备

请先确认电脑已经安装 Node.js 和 npm。

推荐环境：

```bash
node -v
npm -v
```

建议使用 Node.js 20 或更高版本。

## 2. 安装依赖

进入项目目录：

```bash
cd /Users/owenzhao/partTimeA/学习通插件比赛5.20
```

安装依赖：

```bash
npm install
```

说明：交付压缩包默认不包含 `node_modules`，因此首次运行需要执行一次 `npm install`。

## 3. 启动项目

启动前端和后端：

```bash
npm run dev
```

正常启动后会看到类似信息：

```text
Chaoxing lite API listening on http://localhost:3001
VITE ready
Local: http://localhost:5173/
```

如果 `5173` 端口被占用，Vite 会自动切换到 `5174` 或其他端口。以终端实际显示的 `Local` 地址为准。

浏览器打开：

```text
http://localhost:5173/
```

或终端显示的实际地址，例如：

```text
http://localhost:5174/
```

## 4. 首页登录

打开页面后先进入首页：

- 左侧显示“大学英语”标题和平台简介。
- 右侧为登录面板。
- 可选择三类演示账号：
  - 管理员账号：admin / admin123
  - 教师账号：teacher01 / teacher123
  - 学生账号：student04 / student123

选择账号后点击登录，即可进入对应端页面。

## 5. 快速演示路线

推荐按下面顺序演示，能完整体现“管理员建课、学生入班学习、教师发布作业和分析”的闭环。

### 5.1 管理员演示

1. 打开首页。
2. 选择“管理员账号 · admin”并登录。
3. 查看“仪表盘”，确认课程完整度、资源数量、账号数量。
4. 点击左侧“课程管理”。
5. 查看章节结构：Unit、Session。
6. 点击“新建 Unit”可增加课程单元；在每个 Unit 的名称输入框中修改标题后点击“保存 Unit”；不需要的 Unit 可点击“删除 Unit”。
7. 点击任意已有章节标题，进入下方“章节独立编辑页”。编辑器会自动回显该章节已有正文，方便继续修改。
8. 在编辑页中修改章节标题、简介和预计时长，点击“保存章节”。
9. 在富文本编辑区中直接输入正文，使用工具栏设置标题、字号、加粗、斜体、下划线、列表、对齐、链接，并插入图片、音频或视频。
10. 点击“保存到章节正文”或“保存正文修改”，课程浏览页会在资源附件上方展示排版后的章节正文；再次编辑同一章节时会覆盖更新原正文，不会只新增重复内容。
11. 如需删除章节，点击编辑页中的“删除章节”。该章节绑定的资源会一并删除。
12. 在“上传课程资源”区域先选择“绑定章节”，避免资源默认上传到错误章节。
13. 选择章节后，下方会显示该章节当前已绑定资源数量和资源列表；不需要的资源可在列表中点击“删除”。
14. 点击“选择本地文件”，选择 PPT、音频、PDF 或压缩包。
15. 点击“上传并绑定章节”。
16. 页面顶部出现成功提示，课程资源数量会增加。PPT 会显示所选原文件，不再替换成默认 PPT；没有资源的章节会显示“请选择一个资源”，不会自动塞入默认 PPT。
17. 点击左侧“题库管理”。
18. 在“题库管理”页点击题型按钮维护单选题、多选题、填空题、阅读理解、写作题、判断题、简答题和主观题。切换题型后，编辑区会显示对应的选项、填空答案、判断答案、阅读小题或评分要点。
19. 在题目编辑区点击“插入图片/音频/视频”，可给题目绑定媒体，学生考试页会展示图片并可播放音频、视频。
20. 阅读理解题支持“添加小题”和删除小题，可分别维护小题题干、选项、答案和解析。
21. 点击“新建空白题”会清空当前题干、答案、解析和标签，适合连续录入新题。
22. 在“批量导入题库”区域粘贴或上传 TXT/CSV，格式为“题型|题干|选项1;选项2|答案|解析|标签1,标签2”，点击“导入题库”。
23. 切换到“组卷”，可按题型、难度筛选题目，并按默认顺序、题型、难度或题干排序。
24. 从现有题库勾选题目，左侧“当前试卷预览”会实时展示组卷结构；点预览中的移除按钮可删除错选题目。
25. 填写试卷名称、总分和难度，保存到试卷库。
26. 切换到“试卷库”，可修改、删除试卷，也可选择班级后发放试卷。
27. 回到“课程管理”，切换到“账号与班级管理”。
28. 在“学生管理 / 教师团队管理”之间切换，查看成员表格。
29. 使用“排序”下拉框按学号/工号、姓名、班级或加入时间排序。
30. 点击成员行，右侧会显示账号详情。
31. 在“添加成员”表单中创建教师或学生账号。

管理员端重点说明：

- 课程资料只能由管理员上传。
- 教师和学生没有课程资源上传入口。
- 支持 PPT、视频、音频、PDF、资料包等类型。
- 可创建、重命名、编辑和删除课程章节。
- 可创建、重命名和删除 Unit，删除 Unit 时会同步移除其下属章节和资源。
- 可用富文本编辑器维护章节正文，正文支持文字格式、图片、音频和视频；插入媒体后可继续输入普通正文，不会继承图片说明文字样式。
- 富文本编辑器支持继续编辑已有章节正文；删除音频后可再次插入同一音频文件。
- 可查看并删除每个章节已绑定的资源，课程资源页中的默认资源也可由管理员删除。
- 可维护题库，支持多题型按钮切换、题目媒体、阅读小题、单题编辑、删除、新建空白题和 TXT/CSV 批量导入。
- 可按题型和难度筛选题目后组卷，支持预览当前试卷、移除错选题、修改试卷、删除试卷，并从试卷库发放到班级。
- 可全局创建和查看教师/学生账号，并在学生管理中查看所属班级。

### 5.2 学生演示

1. 回到首页或顶部切换到“学生”。
2. 默认学生账号为“周同学”，初始未加入班级。
3. 查看学生仪表盘：学习进度、待完成任务、最近得分。
4. 点击左侧“我的任务”。
5. 在“加入班级”中输入邀请码：

```text
ENG2026A
```

6. 点击“加入”。
7. 加入成功后，页面显示“已加入：大学英语 1 班”。
8. 待完成作业列表出现教师发布的任务。
9. 选择作业，在线答题。
10. 点击“提交试卷”，在确认框中点击“确认提交”。
11. 如果作业只包含单选、多选、填空、判断和阅读理解，系统会自动批改并立即显示分数、答案和解析。
12. 如果作业包含简答、写作或主观题，提交后学生端显示“等待教师批改试卷”，暂不显示分数、答案和解析。
13. 教师批改完成后，学生端会显示最终分数、你的答案、参考答案和解析。
14. 点击左侧“课程资源”。
15. 在左侧课程目录中切换不同章节。
16. 查看 PPT、视频、音频、PDF、资料包等资源。
17. 点击“总结本节”或“自测题”，查看 AI 助教回复。

学生端重点说明：

- 学生可以浏览公开课程资料。
- 学生不能直接访问题库。
- 学生只能加入一个班级，已有班级时不会再显示加入入口。
- 学生通过教师发布的作业/考试进行在线作答。
- 判断题以“正确 True / 错误 False”单选形式作答。
- 提交前会二次确认；提交后不能重新作答。
- 客观题会自动批改；简答、写作和主观题会等待教师批改后再回返成绩。
- 填空题答题区会提示“每个空按顺序填写，用英文逗号分隔”。
- AI 助教会围绕当前课程内容进行总结和答疑。

### 5.3 教师演示

1. 回到首页或顶部切换到“教师”。
2. 点击左侧“班级作业”。
3. 查看“我的班级”和邀请码。
4. 点击某个班级卡片中的“设为发布班级”。再次点击同一按钮可取消当前发布班级。
5. 点击左侧“题库管理”。
6. 在“组卷”中按题型、难度筛选并排序题目，勾选后保存为试卷。
7. 在“试卷库”中选择发放班级，点击“发放”，学生端会收到试卷任务。
8. 回到“班级作业”，也可以在“从题库发布作业”中临时选择题目发布作业；长题干会以两行摘要展示，阅读理解不会撑开整个页面。
9. 页面顶部出现发布成功提示，作业列表数量增加。
10. 学生提交含简答、写作或主观题的试卷后，回到“班级作业”页查看“待教师批改”。
11. 在待批改卡片中查看学生答案、参考要点和解析，为每道主观题输入 0-100 分，点击“提交批改并回返成绩”。
12. 批改完成后，成绩会回到学生端；待批改列表中不再显示该提交。
13. 在“班级学生管理”中查看学生列表。
14. 点击“移出”可以将学生从当前教师班级中删除。
15. 在“AI 教学助手”中点击：
   - 生成思政案例
   - 智能出题
   - 学情讲评建议
16. 点击左侧“学情分析”，查看资源完成情况、风险学生和 AI 教学建议。

教师端重点说明：

- 教师可以创建班级、使用邀请码邀请学生。
- 教师可以从试卷库发放试卷，也可以选择具体班级和具体题目发布作业。
- 教师可以将学生从自己管理的班级中移出。
- 单选、多选、填空、判断和阅读理解提交后自动批改。
- 简答、写作和主观题不会自动按参考答案给分，会回到教师端批改，教师完成后再把总分回返学生端。
- 学情分析基于资源浏览、提交记录和成绩数据。

## 6. 页面功能说明

### 仪表盘

不同角色看到不同信息：

- 管理员：课程完整度、课程资源、账号总数。
- 教师：班级数量、已发布任务、平均成绩。
- 学生：学习进度、待完成任务、最近得分。

### 课程资源

课程资源页采用“课程目录 + 章节正文 + 资源附件预览 + AI 助教”的结构。

支持资源类型：

- PPT：在线翻页，可点击“播放/下一页”。
- 视频：播放器样式和进度条。
- 音频：原生音频播放器，可播放本地示例音频。
- PDF：在线预览样式。
- 资料包：真实下载入口，点击下载 zip。

章节正文：

- 管理员在章节编辑页保存的富文本正文会优先显示在页面上方。
- PPT、音频、PDF 和资料包等附件显示在章节正文下方。
- 图片可点击放大查看。
- 音频和视频使用浏览器原生播放器播放。
- 上传的 PPT 原文件会保留文件名和下载入口，不会再显示成固定默认 PPT。

### 课程管理

仅管理员可见。

包含：

- 章节结构。
- 章节独立编辑页：重命名、保存简介、删除章节、富文本编辑正文、插入图片/音频/视频。
- 资源上传。
- 上传前可选择目标章节。
- 题库编辑/上传：手动编辑题目、删除题目、TXT/CSV 批量导入。
- 发布前检查。
- 账号管理。
- 学生管理表格包含班级列，并支持排序。
- 资源类型统计。

### 题库管理

管理员和教师可见。

包含：

- 题库管理：支持单选题、多选题、填空题、阅读理解、写作题、判断题、简答题和主观题。
- 题目媒体：手动编辑题目时可插入图片、音频和视频，学生端答题页会展示或播放。
- 阅读理解：支持维护多个小题，每个小题有独立题干、选项、答案和解析。
- 新建空白题：清空当前编辑区并保留当前题型，便于连续录题。
- 批量导入：支持按行导入 TXT/CSV 题目，推荐格式为“题型|题干|选项1;选项2|答案|解析|标签1,标签2”。
- 组卷：从题库中按题型、难度筛选题目并排序后组合试卷，实时预览当前试卷并可移除错选题目，保存总分和难度。
- 试卷库：查看已有试卷，支持修改、删除和发放到班级。

### 班级作业

仅教师可见。

包含：

- 班级列表。
- 邀请码。
- 作业/考试列表。
- 发布作业表单：选择班级、选择题目、确认发布。
- 待教师批改：浏览简答、写作和主观题答案，填写分数并回返学生端。
- 班级学生管理：查看学生、移出学生。
- AI 教学助手。

### 我的任务

仅学生可见。

包含：

- 待完成任务。
- 在线作答。
- 客观题自动批改结果。
- 含主观题试卷的“等待教师批改试卷”状态。
- 个人提交记录。

### 学情分析

管理员和教师可见。

包含：

- 活跃学生。
- 平均进度。
- 平均成绩。
- 资源完成情况。
- 风险学生。
- AI 教学建议。

学生访问时只显示个人权限说明，不能查看全班学情。

## 7. API 验证

后端 API 地址：

```text
http://localhost:3001/api
```

健康检查：

```bash
curl http://localhost:3001/api/health
```

学生访问题库会被拒绝：

```bash
curl -H "x-user-id: u-student-chen" "http://localhost:3001/api/questions?courseId=course-ai"
```

预期返回：

```json
{
  "error": {
    "code": "FORBIDDEN",
    "message": "当前角色 student 无权访问该接口"
  }
}
```

管理员查看课程资源：

```bash
curl -H "x-user-id: u-admin" "http://localhost:3001/api/resources?courseId=course-ai"
```

教师查看作业：

```bash
curl -H "x-user-id: u-teacher-lin" "http://localhost:3001/api/assignments?courseId=course-ai"
```

## 9. 交付内容说明

压缩包中主要包含：

- `src/`：React 前端源码。
- `server/`：Express 本地 API。
- `public/`：静态资源。
- `docs/使用文档.md`：本文档。
- `package.json`、`package-lock.json`：依赖和脚本。
- `dist/`：已构建的前端产物。
- `大致需求.docx`、`课程资料结构补充.docx`：原始需求文档。

不包含：

- `node_modules/`：依赖目录体积较大，交付后通过 `npm install` 生成。

## 10. 当前实现边界

这是比赛演示级全栈原型，重点是完整流程和角色权限。

当前 AI 助教已通过后端接口接入真实 DeepSeek API。请把 DeepSeek API Key 放在项目根目录的 `apikey.txt` 中，或用环境变量启动：

```bash
DEEPSEEK_API_KEY=你的key npm run dev
```

后端会优先读取环境变量，其次读取 `apikey.txt`。API Key 只在服务端读取，不会进入前端打包产物。DeepSeek 不可用或未配置 key 时，系统会回退到本地回复，保证演示流程不中断。

线上后端已部署到 Vercel：

```text
https://chaoxing-lite-course-api.vercel.app/api
```

GitHub Pages 前端通过 `VITE_API_BASE_URL=https://chaoxing-lite-course-api.vercel.app/api` 连接该后端，由 Vercel 环境变量 `DEEPSEEK_API_KEY` 安全保存真实 Key。公开 Pages 版不会携带或暴露 API Key，但可以通过线上后端调用真实 DeepSeek。

实际接入位置：

```text
server/index.ts
```

中的：

```text
POST /api/ai/assistant
```

真实账号登录、数据库持久化也可以在现有 API 和数据模型基础上继续扩展。当前资源上传在本地演示环境中会把所选文件读取为 data URL 并提交到本地 API，适合比赛演示和小型文件验证；生产环境建议替换为对象存储或后端 multipart 上传。
